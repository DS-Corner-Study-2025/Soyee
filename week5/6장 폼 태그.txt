6장 폼 태그

1. 폼 태그 개요
- 사용자 -> 웹 요청 URL -> 폼 페이지 -> 웹 서버로 전달
- GET 방식 : 폼 입력 페이지
- POST 방식 : 응답 웹 페이지
- <form 속성=' '> <input>, <select>, <textarea> </form>

2. 데이터 바인딩 (@ModelAttribute)
- 커맨드 객체 : 요청 파리미터를 프로퍼티로 데이터를 바인딩함
- 롬복 : Getter(), Setter() 사용할 필요없이 자동생성. 의존성 주입 필요
- @ModelAttribute : 폼페이지 데이터 전달되면  -> 커맨드 객체에 매핑 -> 데이터 프로퍼티에 채움
- 매서드 수준의 @ModelAttribute

3. 커스텀 데이터 바인딩 (@InitBinder)
- @InitBinder : 매핑되기 전에 데이터 바인딩을 사용자 정의, WebDataBinder 객체 초기화하는 매서드와 함께 사용
- SetAllowedFields() : 데이터 바인딩 허용
- SetDisallowedFields() : 데이터 바인딩 허용 X

###빈칸 문제
1. <form> 태그에서 데이터를 전송하는 HTTP 방식을 설정하는 속성은 _____ 입니다. (예: GET, POST) -------> method
2. <select> 태그의 하위 태그인 <option>에서, 해당 항목을 초깃값으로 선택되도록 설정하는 속성은 _____ 입니다. --------> selected
3. 여러 줄의 텍스트를 입력할 수 있는 태그는 _____ 입니다. --------> <textarea>
4. 스프링에서 폼 데이터를 커맨드 객체의 필드에 자동으로 바인딩하고 Model에도 추가하는 역할을 하는 어노테이션은 _____ 입니다. --------> @ModelAttribute
5. @InitBinder를 사용하여 특정 폼 파라미터(예: hobby)가 데이터 바인딩되는 것을 차단할 때 사용하는 WebDataBinder의 메서드는 _____ 입니다. ----------> SetDisallowedFields()

문제 2: @ModelAttribute 사용
Spring Controller에서 폼으로부터 전송된 데이터를 Member 객체로 바인딩하고, 이 객체를 "memData"라는 이름으로 Model에 추가하는 요청 처리 메서드를 완성하세요.

@PostMapping("/member")
public String submitForm((A) Member member, Model model) {
    // Member 객체를 "memData"라는 이름으로 Model에 추가
    model.addAttribute("memData", (B)); 
    return "resultView";
}
------> (A) @ModelAttribute
------> (B) member
