8장 다국어 처리

1. 다국어 처리 개요
- 로케일에 따라 다양한 언어 지원
- 로케일 : 지역별 설정을 정의하는 문자열 (언어, 글자, 날짜, 시간 표시 양식 등등)
- 국제화 -> 코드변경X, 별도 페이지X 

2. 다국어 처리 

- 메세지 리소스 파일 (resource 파일 안에 존재)
message.properties 
message_ko.properties - 한국어
message_en.properties - 영어
message_en_US.properties - 영어(영국)
message_ja.properties - 일본어

- 메세지 리소스 파일 내용
Person.form.Enter.message = value

- MessageSource 인터페이스 구현체
(1) ResourceBundleMessageSource : 자동으로 빈 객체 등록되어있음 -> 메시지 리소스 파일에서 가져올 수 있음
(2) ReloadableResourceBundleMessageSource : 실행 도중에 메세지 다시 로드 가능

- MessageSource 메세지 출력
<span th:text="#{Person.form.Enter.message}"/>

- 사용자 정의 환경 설정
- application.properties
- application.yml
- spring.message.basename = message/i18n

3. 다국어 변경
- 로케일 설정
- LocaleResolver
- 구현체 유형 : AcceptHeader, Cookie, Session, Fixed

- 로케일 변경
- LocaleChangeInterceptor
- 컨트롤러 구현X




###빈칸 문제
1. 다국어 처리에서 언어별 메시지는 보통 key=value 쌍으로 작성된 _____ 확장자를 가진 리소스 파일에 저장됩니다. -----------> properties
2. Spring Boot는 _____ 인터페이스를 통해 메시지 리소스 파일에서 언어별 메시지를 가져와 출력하는 기능을 제공합니다. -----------> MessageSource
3. 웹 브라우저의 _____에 따라 한국어 메시지를 출력하려면, 메시지 리소스 파일 이름을 messages_ko로 만들고 파일 확장자는  .properties로 작성해야 합니다. -----------> 로케일
4. 사용자 정의 환경 설정을 위해 메시지 리소스 파일의 기본 이름 및 위치를 설정하는 속성은 spring.messages._____ 입니다. -----------> basename
5. 로케일 정보(예: ko_KR)를 사용자 세션(Session)에 저장하여 언어를 유지하게 하는 로케일 결정 유형은 _____ 입니다. --------------> SessionLocaleResolver
6. LocaleResolver 중, HTTP 요청 헤더의 Accept-Language 값을 이용하여 로케일을 결정하는 유형은 _____ 입니다. ---------->  AcceptHeaderLocaleResolver 
7. application.yml 파일에서 메시지 리소스 파일의 인코딩을 지정할 때 사용하는 속성은 spring.messages._____ 입니다. ----------------> encoding

public LocaleResolver localeResolver() {
